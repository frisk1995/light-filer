# filox

## 概要
**filox** は Windows 向けの高速・軽量ファイルマネージャーである。Rust + egui（eframe）で実装されたネイティブデスクトップアプリで、Windows 標準エクスプローラーの代替として、起動の速さ・スキャンの速さ・落ち着いた配色を重視している。対象ユーザーは開発者・パワーユーザーで、ドライブ閲覧、ファイル操作、クイックアクセス、OneDrive 連携、ターミナル起動などを 1 つのウィンドウで完結させたい人を想定する。

- **目的:** Windows のファイル閲覧・基本操作を、軽快な独自 UI で行う
- **対象 OS:** Windows（`#[cfg(windows)]` で WinAPI を直接呼び出す。非 Windows ではビルドのみ通る縮退動作）
- **技術スタック:** Rust 2021 / eframe 0.29（glow バックエンド）/ egui 0.29 / egui_extras / crossbeam-channel（FS ワーカースレッド通信）/ chrono（更新日時）/ dirs（標準フォルダ解決）/ open（既定アプリ起動）
- **バージョン:** 0.4.7
- **ウィンドウ:** 初期サイズ 1200×760、最小サイズ 640×480、タイトル "filox"

> 注: このドキュメントは既存コードから逆生成した「現状の実装仕様」である。各スプリントの受け入れ基準は Evaluator が現行ビルドに対して合否判定できる粒度で記述している。

## 画面構成

アプリは上下のパネルと中央コンテンツで構成される。上から順に以下。

### 1. タイトルバー（高さ 38px）
- 左端に角丸の「f」マーク（アクセントカラー背景・白文字）
- アプリ名 "filox"（太字・等幅）とバージョン "0.4.7"（淡色）
- 中央に現在のフルパスを等幅・淡色で表示

### 2. ツールバー（高さ 46px）
- **左側ナビゲーションボタン群:** 戻る（◁）／進む（▷）／上へ（△）／更新（⟳）／パスを開く（Ctrl+L、ホバーで「パスを開く (Ctrl+L)」ツールチップ）
  - 戻る・進むは履歴が空のとき無効表示
- **右側:**
  - 設定ボタン（歯車。トグルでパネル開閉）
  - ビューモード切替セグメント（リスト＝Explorer／グリッド＝Modern）
  - 検索ボックス（プレースホルダ "Search…"）
  - パンくずリスト（パス各階層をクリックで移動。階層が 4 を超えると先頭を "…" に省略）

### 3. ステータスバー（高さ 27px、最下部）
- 左: アイテム数、選択数＋選択合計サイズ、タグ数＋タグ合計サイズ（`N items · M selected · X · K tagged · Y`）
- ファイルオープン中はスピナーと「「<名前>」を開いています...」を表示（5 秒で自動消去）
- 右: 空き容量「<n> GB free」とスキャン時間「⚡ indexed in <ms> ms」（スキャン中は "Scanning…"）

### 4. 左ペイン（サイドバー）
**Explorer ビュー（幅可変 140–480px、初期 222px）:**
- `QUICK ACCESS`: 登録済みクイックアクセス項目（既定 Home / Downloads / Desktop）。右クリックで「Remove from Quick Access」
- `CLOUD`: OneDrive が検出された場合のみ表示（個人用・ビジネス用）。クラウドアイコンは青
- `THIS PC`: 検出された全ドライブ。各ドライブは使用量バー（90% 超で赤）と「<free> GB / <total> GB」を表示
- `FOLDERS`: 現在パスの祖先〜現在ディレクトリをツリー状に表示。右クリックで「クイックアクセスに追加」。ドロップ先になる

**Modern ビュー（幅固定 190px）:**
- `QUICK ACCESS`: "Projects"（C:\dev）/ "Recent"（C:\）の固定項目
- 現在フォルダ名の見出し下に、現在ディレクトリ内のサブフォルダ一覧

### 5. 中央コンテンツ
**Explorer ビュー（リスト）:**
- 列: Name / Size / Type / Modified
- ヘッダーの Name|Size 境界をドラッグして Name 列幅を変更可能
- Name 列が広いと水平スクロールバーが出る／行は垂直スクロール
- 各行: アイコン＋名前（ピクセル単位で省略 "…"）＋クラウドバッジ＋サイズ（右寄せ）＋種別＋更新日時

**Modern ビュー（グリッド）:**
- タイル（110×90px）をウィンドウ幅に応じて自動列数で並べる
- 各タイル: 大アイコン＋名前（14 文字超で省略）
- 右側に幅 300px のプレビューパネル（選択ファイルの情報・テキスト先頭 20 行プレビュー・Open / More ボタン）

## スプリント計画

> 既存実装に対する受け入れ基準である。各項目は現行ビルドで合否判定可能なこと。

### Sprint 1: 起動とディレクトリ表示
**ゴール:** アプリが起動し、ディレクトリ内容を正しく一覧表示できる
**機能:**
- [ ] ディレクトリスキャン（バックグラウンドスレッド）
- [ ] 一覧表示（4 列）とソート
- [ ] ステータスバーのアイテム数・スキャン時間表示

**受け入れ基準:**
- アプリ起動時、ウィンドウタイトルが "filox" でサイズ 1200×760 程度、左上に "filox" と "0.4.7" が表示される
- 起動時に作業ディレクトリ（取得失敗時は `C:\`）の内容が表示される
- 一覧はフォルダが上、ファイルが下に並び、各グループ内は名前の昇順（大文字小文字無視）でソートされる
- 各行に アイコン・名前・サイズ（B/KB/MB/GB）・種別ラベル・更新日時（`YYYY/MM/DD HH:MM`）が表示される
- フォルダ行のサイズ欄は空、ファイルのサイズはバイト数に応じた単位で表示される
- ステータスバー左に「<N> items」、右に「<free> GB free ⚡ indexed in <ms> ms」が表示される
- 読み取り権限のないフォルダへ移動してもクラッシュせず、空一覧になる

### Sprint 2: ナビゲーション
**ゴール:** フォルダ間を自在に移動できる
**機能:**
- [ ] 戻る/進む/上へ/更新ボタン
- [ ] パンくず移動
- [ ] パス直接入力（Ctrl+L）
- [ ] マウスサイドボタン

**受け入れ基準:**
- フォルダをダブルクリックするとそのフォルダ内へ移動し、タイトルバー・パンくずのパスが更新される
- 「上へ」ボタンで親フォルダへ移動する
- 「戻る」で直前のフォルダへ、「進む」で戻る前のフォルダへ移動する
- 履歴が無いとき「戻る」「進む」ボタンは無効（淡色）表示になる
- 新しい場所へ移動すると「進む」履歴はクリアされる
- パンくずの中間階層をクリックするとその階層へ移動する
- パス階層が 5 段以上のとき、先頭が "…" に省略される
- Ctrl+L または「パスを開く」ボタンでダイアログが開き、現在パスが初期表示される
- ダイアログに有効なフォルダパスを入れて Enter/「開く」で移動、ファイルパスなら既定アプリで開く
- 存在しないパスを入れると「パスが見つかりません」が赤字表示され、ダイアログは閉じない
- Esc またはキャンセルでダイアログが閉じる
- マウスのサイドボタンで戻る/進むが機能する

### Sprint 3: サイドバー（クイックアクセス・ドライブ・OneDrive・フォルダツリー）
**ゴール:** 主要な場所へワンクリックで移動できる
**機能:**
- [ ] クイックアクセス（既定 3 項目＋追加/削除＋永続化）
- [ ] ドライブ一覧と使用量バー
- [ ] OneDrive 検出
- [ ] フォルダツリー（祖先表示）

**受け入れ基準:**
- 初回起動時、QUICK ACCESS に Home / Downloads / Desktop が表示される
- クイックアクセス項目をクリックすると該当フォルダへ移動し、現在地の項目はアクセント色で強調される
- フォルダを右クリック→「クイックアクセスに追加」で項目が追加され、重複追加はされない
- クイックアクセス項目を右クリック→「Remove from Quick Access」で削除される
- 追加/削除はアプリ再起動後も保持される（`quick_access.txt` に永続化）
- THIS PC に検出済みドライブが列挙され、各ドライブに使用量バーと「<free> GB / <total> GB」が表示される
- ドライブ使用率が 90% を超えるとバーが赤色になる
- OneDrive 環境変数が設定されている場合のみ CLOUD セクションが表示され、クリックで移動できる
- FOLDERS に現在パスの祖先〜現在フォルダがツリー表示され、現在フォルダは FOLDER_OPEN アイコン＋アクセント強調になる

### Sprint 4: ファイル操作（作成・リネーム・削除・コピー・ターミナル）
**ゴール:** 基本的なファイル操作を一通り実行できる
**機能:**
- [ ] 新規フォルダ/新規テキストファイル作成
- [ ] リネーム（F2 / メニュー / 作成直後）
- [ ] 削除（確認ダイアログ）
- [ ] パスをコピー
- [ ] ターミナルで開く

**受け入れ基準:**
- 一覧の空白部分を右クリック→「新規作成」→「フォルダー」で「新しいフォルダー」が作成され、即座にインライン名前編集状態になる
- 同名が存在する場合「新しいフォルダー (2)」のように連番が付与される
- 右クリック→「新規作成」→「テキスト ドキュメント (.txt)」で空の .txt が作成され、即リネーム状態になる
- 項目を選択して F2、または右クリック→「名前の変更」でインライン入力欄が表示される
- 名前を入力して Enter で確定し実ファイルがリネームされる、Esc またはフォーカス喪失でキャンセルされる
- 空名・既存名と同一の場合はリネームされない
- 右クリック→「削除」で「削除の確認」ダイアログが中央表示され、「この操作は元に戻せません。」が出る
- 複数選択中に削除すると「<N>個のアイテムを削除しますか？」、単一なら「「<名前>」を削除しますか？」が出る
- 「削除」確定でファイル/フォルダ（フォルダは中身ごと）が削除され、一覧が更新される。「キャンセル」で何も起きない
- 右クリック→「パスをコピー」でフルパスがクリップボードに入る
- フォルダを右クリック→「ターミナルで開く」で Windows Terminal（無ければ PowerShell）がそのフォルダで開く

### Sprint 5: 選択・検索・隠しファイル
**ゴール:** 一覧を絞り込み・選択できる
**機能:**
- [ ] 単一/複数選択
- [ ] 検索フィルタ
- [ ] 隠しファイル表示切替

**受け入れ基準:**
- 行をクリックすると単一選択され、行がアクセント色＋左端バーで強調される
- Ctrl または Shift を押しながらクリックで複数選択がトグルされる
- 複数選択中、ステータスバーに「<M> selected · <合計サイズ>」が表示される
- 検索ボックスに文字を入れると、現在フォルダ内で名前に部分一致（大文字小文字無視）する項目だけ表示される
- フォルダ移動すると検索文字列はクリアされる
- 設定で「隠しファイルを表示」をオフにすると `.` 始まりの項目が一覧から消え、オンで再表示される
- 隠し項目は淡色で表示される

### Sprint 6: ドラッグ＆ドロップ移動
**ゴール:** ドラッグでファイルを別フォルダへ移動できる
**機能:**
- [ ] 行/タイルのドラッグ
- [ ] フォルダ・サイドバーフォルダツリーへのドロップ

**受け入れ基準:**
- 項目をドラッグするとカーソル近くにファイル名（複数時は「<N> 個のアイテム」）のフローティングラベルが出る
- 未選択の項目をドラッグ開始するとその項目が単独選択に切り替わる
- ドロップ先候補のフォルダ行/サイドバーのフォルダツリー項目に、ドラッグ中アクセント色の枠線が表示される
- フォルダにドロップするとドラッグ中の全ファイルがそのフォルダへ移動し、一覧が更新される
- 同一フォルダへのドロップ（移動先＝現在地）では何も起こらない

### Sprint 7: ビューモード・プレビュー
**ゴール:** リスト/グリッドを切替え、ファイルをプレビューできる
**機能:**
- [ ] ビューモード切替（Explorer / Modern）
- [ ] Modern グリッド表示
- [ ] プレビューパネル

**受け入れ基準:**
- ツールバーのセグメントでリスト（Explorer）/グリッド（Modern）を切替えられる
- Modern ビューでファイルがタイル状（大アイコン＋名前、14 文字超は省略）に並ぶ
- Modern ビューでタイルをクリックすると右側プレビューパネルにアイコン・名前・パス・Type/Size/Modified が表示される
- テキスト系ファイル（rs/toml/md/txt/json/yaml/yml/lock）はプレビューパネルに先頭 20 行が表示される
- プレビューの「Open」ボタンでファイルが既定アプリで開く
- Modern ビューでフォルダタイルをダブルクリックするとフォルダ内へ移動する
- 未選択時はプレビューパネルに「Select a file to preview」が表示される

### Sprint 8: テーマ・フォント・列幅（外観設定）
**ゴール:** 外観を好みに調整でき、設定が永続化される
**機能:**
- [ ] テーマ切替（ダーク/ライト）
- [ ] フォント切替（6 種）
- [ ] Name 列幅リサイズ

**受け入れ基準:**
- 設定ボタンで設定パネルが右上に開閉し、✕ で閉じる
- 「ダーク」「ライト」を切替えると全体の配色が即座に変わる
- フォント 6 種（メイリオ / Yu Gothic / BIZ UD Gothic / MS Gothic / Noto Sans JP（内蔵）/ IBM Plex Sans（内蔵））を選択でき、選択が即反映される
- テーマ・隠しファイル設定・フォントはアプリ再起動後も保持される（`settings.txt` に永続化）
- Explorer ヘッダーの Name|Size 境界をドラッグすると Name 列の幅が変わり、必要に応じ水平スクロールが出る

## UI/UX 要件

- **配色トークン:** ダーク/ライトの 2 テーマ。背景・タイトルバー・リスト・境界・ホバー・浮き上がり・本文・淡色・極淡色を `Tokens` で集中管理。アクセントは Rust（橙 `#e0824a`）固定（コードには Aqua/Green/Purple も定義あり）
- **アイコン:** Material Symbols Rounded のコードポイントを内蔵フォントで描画。フォルダは黄系、ファイル種別ごとに色分け（rs=アクセント、toml=緑、md=ティールなど）
- **画面遷移:** 単一ウィンドウ内でビュー（Explorer/Modern）を切替。ダイアログ（削除確認・パス入力）と設定パネルはモーダル風オーバーレイ
- **主要ユーザーフロー:**
  1. 起動 → サイドバーから場所選択 → 一覧表示 → ダブルクリックで掘り下げ
  2. 右クリック → 新規作成/リネーム/削除/コピー/ターミナル
  3. ドラッグ＆ドロップで別フォルダへ移動
  4. 検索ボックスで絞り込み
- **キーボード:** Ctrl+L（パス入力）、F2（リネーム）、Enter（確定）、Esc（キャンセル）
- **フィードバック:** ファイルオープン時スピナー、ドラッグ中フローティングラベル、ドロップ先ハイライト、ドライブ使用量バー

## 設定・永続化

設定ファイルは `dirs::config_dir()/filox/`（Windows では `%APPDATA%\filox\`）配下に保存される。

| ファイル | 形式 | 内容 |
|----------|------|------|
| `settings.txt` | `key=value` 行 | `theme=dark\|light`、`show_hidden=true\|false`、`font=<id>`（既定: dark / false / notosansjp） |
| `quick_access.txt` | `name<TAB>path` 行 | クイックアクセス項目（名前とパス） |

- クイックアクセスは追加/削除のたびに即保存される
- テーマ・隠しファイル・フォントは設定変更のたびに即保存される
- クイックアクセスファイルが空の場合は既定の Home/Downloads/Desktop を使用する

## 既知の制限事項

- **Windows 専用機能:** ドライブ列挙・容量取得・クラウド専用ファイル検出は WinAPI（`GetLogicalDrives` / `GetDiskFreeSpaceExW` / ファイル属性）に依存。非 Windows ではドライブは `/` 1 件・容量 0 の縮退動作
- **コピー/切り取り/貼り付け非対応:** ファイル操作は移動（ドラッグ＆ドロップ／rename）と削除・作成・リネームのみ。クリップボード経由のコピー＆ペーストや異なるドライブ間移動（rename はクロスデバイス不可）は未実装
- **削除はゴミ箱を経由しない:** `remove_file` / `remove_dir_all` による完全削除で、元に戻せない
- **新規ファイルテンプレートは .txt のみ:** UI 上のテンプレートはテキストドキュメントのみ（内部的に rtf は空ヘッダ生成に対応）
- **ソート列固定:** 内部に Size/Kind/Modified のソート列定義はあるが、UI からの並べ替え変更は未配線（常に名前昇順）
- **タグ機能未配線:** ステータスバーにタグ数表示のロジックはあるが、タグ付け操作は UI 未実装
- **検索はカレントフォルダのみ:** サブフォルダ再帰検索は行わず、現在フォルダ内の名前部分一致のみ。Modern ビューには検索フィルタが適用されない
- **エラーは握りつぶし:** ファイル操作（作成/リネーム/削除/移動）失敗時の明示的なエラーダイアログは無く、失敗は無視される（スキャン失敗時はログ出力）
- **Modern ビューのサイドバー一部固定パス:** "Projects"=`C:\dev`、"Recent"=`C:\` はハードコード
- **アクセントカラー選択 UI 無し:** 内部に 4 色定義があるが UI からは Rust 固定
